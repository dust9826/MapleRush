0. 한눈에 보기

장르: 1인 1런 로그라이트. 한 맵(/maps/map01, 사이드뷰+풋홀드)에서 스테이지를 갈아끼우는 방식 — 새 맵으로 이동하지 않고 적/투사체를 비우고 재스폰한다(StageManager.ResetStage).

핵심 루프: 테마 선택 → 노드(일반 N회) → 상점 → 보스 → 다음 층 → … → 전층 클리어. 클리어 시 영구 강화 점수 적립(MetaProgression).

전투 한 줄 요약: 좌클릭=평타, 우클릭=대시(조준→실행), 휠클릭=능력, 1/2/3=아이템. 플레이어는 1피격 즉사, 적은 HP제. 대시 시작 시 근처에 적 위협이 있으면 슬로우모션 + 강화대시 1회 보상.

1. 서버 / 클라 권위 모델 (가장 먼저 이해할 것)

이 코드를 읽을 때 "이 메서드는 어느 쪽에서 도느냐"가 전부다. @ExecSpace로 강제된다.

@ExecSpace의미이 프로젝트에서 담당
ClientOnly클라에서만입력 감지, 연출(이펙트/고스트/타임스케일), HUD/팝업 갱신
ServerOnly서버에서만데미지/명중/무적/죽음/스폰/저장 — 모든 권위 판정
Server클라가 부르면 서버로 RPCRequestXxx — 클라 입력을 서버 판정으로 넘기는 관문
Client서버가 부르면 특정 클라로 RPCClientShowXxx — 서버 결과를 한 유저 화면에 반영
Multicast전 클라 동시EnemyHealth.PlayHitEffect (모두에게 피격 이펙트)
읽기 요령: 의심나는 동작은 RequestXxx(클라→서버) / ClientShowXxx(서버→클라) 이름만 따라가면 클라-서버 왕복을 추적할 수 있다. 데미지 숫자가 이상하면 무조건 서버 메서드(CombatPrimitives, RequestAttack, ServerDashStarted)부터 본다.

4. 부팅 & 컴포넌트 부착

유저 입장 (UserEnterEvent) └─ PlayerBootstrap.HandleUserEnter SERVER ├─ MetaProgression.LoadFor(userId) // 영구강화 로드 ├─ AttachCombatComponents(player) │ └─ AddComponent × 6 // Combat/Dash/InputSetup/Hit/Ability/Items ├─ EnsureDashPreview(player) // 조준선 + 목적지 마커 자식 스폰(1회) └─ StageManager.OnPlayerReady(player) └─ GameStateManager.StartRun() + ShopManager.ResetPrices() + FloorManager.StartGameFlow()
왜 런타임 부착? 플레이어 본체는 Global/DefaultPlayer.model(읽기 전용)이라 전투 컴포넌트를 모델에 넣을 수 없다. 그래서 입장 시 AddComponent로 붙인다. → 새 플레이어 능력 추가 시 PlayerBootstrap.AttachCombatComponents에 한 줄 추가하는 패턴.

5. 입력 파이프라인

모든 입력은 InputRouter(@Logic, 클라) 하나를 거쳐 RouterActionEvent(action 문자열)로 방송된다. 각 플레이어 컴포넌트는 로컬 플레이어일 때만 구독.

원시 입력액션구독자 · 동작
좌클릭/터치primaryPlayerCombat 평타 / PlayerItems 활성 슬롯 사용(평타보다 우선)
우클릭 누름 → 해제dashAim → dashExecPlayerDash 조준 시작 → 대시 실행
휠클릭 (Mouse2)abilityPlayerAbility 사용
숫자키 1/2/3item1/2/3PlayerItems 슬롯 토글
G키debugSlowPlayerCombat → 슬로우 강제발동 (테스트용, 출시 제거)
WASD—PlayerInputSetup이 화살표 액션 복사 → MovementComponent 기본 처리
컨텍스트 게이팅: InputRouter.CurrentContext가 "Combat"일 때만 액션 발행. 상점/보상/팝업/게임오버에서는 입력 차단. 새 화면 만들면 SetContext 호출 잊지 말 것.

6. 게임 루프 (층 / 스테이지 상태 머신)

두 매니저가 짝을 이룬다: FloorManager(무엇을 할지 — 테마/노드/상점/보스 분기) ↔ StageManager(전투 한 판 — 스폰/클리어/사망).

FloorManager.StartGameFlow → ShowThemeSelect → [테마 팝업] → RequestSelectTheme → NextNode ──┬─ 일반 N회 미만? → [노드 팝업] → RequestSelectNode → StageManager.StartStage("normal") ├─ 상점 미완료? → StartStage("shop") + [상점 팝업] → RequestLeaveShop └─ 그 외 → StartStage("boss") StageManager.StartStage → RollComposition → ResetStage (적 비우고 재스폰 + 1.5s 진입유예) → 전투 → 적 전멸 시 OnEnemyKilled → StageClear (메소+보상롤) → [보상 팝업] → RequestProceedNext → FloorManager.OnStageRewardDone └─ boss 클리어 & 마지막 층? → GameClear → MetaProgression.AwardScore boss 클리어 & 더 있음? → CurrentFloor++ → ShowThemeSelect normal 클리어? → NormalsCleared++ → NextNode 플레이어 사망: PlayerHit.RequestHit → StageManager.OnPlayerDeath └─ 목숨>0 → 목숨-- + ResetStage(재시작) / 목숨0 → GameOver → [게임오버 팝업]
스테이지 전환의 단일 진실은 StageManager.ResetStage. 적/투사체 소멸, 슬로우/전장효과 해제, 카운터 리셋, 재스폰, 플레이어 위치/능력 리셋이 전부 여기 모여있다. 전환 버그는 여기부터.

7. 전투 판정 파이프라인

모든 데미지/명중은 서버에서 CombatPrimitives를 통해 처리된다. 클라는 연출만.

플레이어 평타 좌클릭 → PlayerCombat.TryAttack CLIENT (쿨/대시중/아이템중 가드) → PlayAttackEffect(연출) + RequestAttack SERVER → CombatPrimitives.DamageEnemiesInCircle // 반경 내 EnemyHealth.TakeDamage → CombatPrimitives.TryParry // 노란 공격 무효 / 투사체 반사 적 공격 (텔레그래프 모델) EnemyMelee.OnUpdate → RegisterEnemyAttack(공격레코드 등록, hitDelay) // 슬로우 트리거·패링 대상 → windup 종료 → CircleOverlap로 명중 확인 → PlayerHit.RequestHit ※ activeAttacks 레코드는 CombatPrimitives.OnUpdate가 만료 정리 투사체 EnemyRanged.Fire / BossController(tripleShot) → SpawnByModelId("enemyprojectile") → Projectile.OnUpdate(서버 직선 시뮬) → 플레이어 명중 시 RequestHit → 패링되면 Reflect() → OwnerSide="player"로 전환되어 적에게 데미지 보스 그로기 EnemyHealth.TakeDamage(boss) → GroggyGauge.Charge → 100% → IsGroggy 3초(완전정지) 근접 패링 시 CombatPrimitives.TryParry가 GroggyGauge.Charge(ATK×2) 직접 호출
핵심 유틸 (CombatPrimitives) — 범위 판정은 전부 이 함수들만 쓴다: CircleOverlap (모든 원형 겹침) DamageEnemiesInCircle DamageEnemiesAlongPath (대시) TryParry / AnyThreatNear RaycastWallDistance (대시 벽정지) DirectionTo / PointSegmentDistance

8. Core (인프라) 상세

파일타입봐야 할 핵심
InputRouter@LogicCLIOnScreenTouch/OnKeyDown/OnKeyUp(입력 분기), Emit(이벤트 발행), CurrentContext(게이팅), CursorWorld
RouterActionEvent@Eventaction/worldPos/context 3필드. 모든 입력 액션의 운반체
WallClock@LogicNow/Elapsed/IsExpired(클라 실제시간), ServerNow/IsServerExpired. delta 누산 금지, 스냅샷 차분만
GameConstants@Logic모든 튜닝값. 스탯/반경/데미지%/타이머/이펙트 RUID/머티리얼명. NormalStagesForFloor/ClampMeso 헬퍼
CombatPrimitives@LogicSRV위 7장 유틸 목록 + RegisterEnemyAttack(공격레코드)·activeAttacks 관리
WallClock가 왜 중요? 슬로우모션이 SetClientTimeScale(0.25)로 클라 시간을 늦추기 때문에, 쿨다운을 delta로 누산하면 망가진다. 그래서 모든 타이머가 IsExpired(anchor, dur)(시점 스냅샷 차분) 형태로 통일됨. 새 타이머 만들 때 이 규율 반드시 따를 것.

13. 반복되는 패턴 (한 번 익히면 전체가 읽힘)

① 로컬 플레이어 게이트 — 클라 컴포넌트 OnBeginPlay 첫 줄:

if self.Entity ~= _UserService.LocalPlayer then return end

② 클라 입력 → 서버 판정 관문 — TryXxx(클라 가드/연출) → RequestXxx(@ExecSpace("Server") 권위 처리).

③ 서버 결과 → 특정 클라 화면 — ClientShowXxx(@ExecSpace("Client")) + InputRouter.SetContext.

④ 스냅샷 타이머 — anchor = WallClock:Now() 저장 후 WallClock:IsExpired(anchor, dur). delta 누산 금지.

⑤ 폴링 게이트 — 적 AI가 매 프레임 StageManager:IsGraceActive()·BattlefieldFx:IsStunActive()·SlowMotion:Scale()를 물어 행동 제어.

⑥ SpriteRUID 교체 애니 — PlayClip(ruid)가 변경 시에만 set(@Sync라 클라 자동 반영). 매 프레임 set하면 0프레임 정지.

⑦ 컴포넌트 조회 결합 — GetComponent("script.Xxx")로 형제 상태 읽기(차단/면제 판정).

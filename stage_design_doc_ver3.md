## 1. 적 종류 및 행동 로직

### 1.1 적 종류 개요

| 분류 | 출현 위치 | 공격 방식 |
|---|---|---|
| 근거리 적 | 주로 일반 스테이지 | 플레이어에게 접근 후 근접 공격 |
| 원거리 적 | 주로 일반 스테이지 | 플레이어 위치를 향해 투사체 발사 |
| 보스 | 보스 스테이지 전용 | 정해진 패턴 풀에서 랜덤 선택; 근거리·원거리 공격 혼용 가능 |

---

### 1.2 근거리 적

#### 1.2.1 기본 행동 로직

| 단계 | 내용 |
|---|---|
| 대기 | 플레이어가 감지 범위 밖이면 제자리 대기 (또는 순찰) |
| 추격 | 플레이어가 감지 범위 내에 진입하면 플레이어 방향으로 이동 |
| 공격 | 플레이어와 근접 공격 범위 내에 들어오면 근접 공격 판정 발생 |
| 복귀 | 플레이어가 감지 범위를 벗어나면 추격을 멈추고 **마지막 위치에서 대기** (원위치 복귀는 미적용) |

#### 1.2.2 공격 색상 구분

- 적의 공격은 **노란색** 또는 **빨간색**으로 구분됨 (플레이어 패링 시스템 연계 → `player_design_doc.md` §5 참조)
- 기본 근거리 공격은 **노란색**(패링 가능)으로 설정하되, 특수 패턴은 **빨간색**(패링 불가)로 설정 가능
- **[코드 구현 — 갭② 해소]** 각 공격/투사체에 **`parryable: boolean`** 플래그 부여 (true=노랑/패링·반사 가능, false=빨강/불가). 색상은 추후 공격 이펙트로 시각화하고, 판정 로직은 boolean으로 분기 (`player_design_doc.md` §5.2 참조)

#### 1.2.3 예외 처리

> - 감지 범위: 플레이어 크기의 **3배(= 플레이어 중심 반지름 3 world unit 원형)** / 공격 범위: 플레이어 크기의 **1배(= 반지름 1 world unit)** / 공격 방향: 플레이어를 향함
> - 플레이어가 감지 범위를 벗어나거나 스테이지 외곽 등 추격 불가 위치에 있을 경우, 적은 **마지막 추격 위치에서 대기** (원위치 복귀는 미적용으로 통일)
> - 동일 프레임에 여러 근거리 적의 공격 판정이 겹쳐도 플레이어 피격 판정은 1회만 적용 (1피격 즉사 원칙 유지)
> - **낭떠러지(플랫폼 끝) 처리**: 적은 낭떠러지로 떨어지지 않는다. 플랫폼 끝에 도달하면 그 자리에서 정지한다. **단 이는 MSW 기본 동작이 아니므로 수동 구현이 필요하다(A-4)** — 적 AI에서 진행 방향 전방 바닥을 `FootholdComponent:Raycast(point, Vector2(0,-1), dist)`(또는 현재/다음 발판의 `Foothold.EndPoint`·`NextFootholdId`)로 검사해 발판 끝이면 이동을 멈춘다. (낙하 물리 자체는 비활성/미구현) ✅ **검증완료(V2 실측)**: 바닥 위 좌표는 `GROUND`(Foothold 반환), 바닥 끝 너머 좌표는 `NONE`을 반환 — Raycast로 발판 끝 감지가 정확히 동작함을 확인.

### 1.3 원거리 적

#### 1.3.1 기본 행동 로직

| 단계 | 내용 |
|---|---|
| 대기 | 플레이어가 감지 범위(플레이어 크기 5배 = 반지름 5 world unit) 밖이면 제자리 대기 |
| 추격(이동) | 플레이어가 감지 범위 내에 진입하면 **근거리 적과 동일하게 플레이어 방향으로 이동** |
| 조준 | 감지 범위 내 플레이어를 향해 조준 시작 |
| 발사 | **조준 시작 후 1초 뒤 투사체 1개 발사**; 발사 후 **1초 쿨타임** 뒤 다시 조준 시작 (감지 범위 내 플레이어 존재 시 반복) |

> 원거리 적은 더 이상 고정형이 아니다. 감지 범위(5배) 안에서 플레이어를 향해 이동하면서 **조준 1초 → 발사(1개) → 쿨타임 1초** 사이클을 반복한다. 1발 사이클당 총 소요 시간은 약 2초이다.

#### 1.3.2 투사체 사양

| 항목 | 내용 |
|---|---|
| 발사 방향 | 발사 시점의 플레이어 위치를 목표로 직선 발사 (유도 없음) |
| 투사체 속도 | **10** |
| 투사체 히트박스 크기 | **플레이어 크기의 1/5배 (= 반지름 0.2 world unit 원형)** (충돌 판정용 크기, 발사 거리 개념 아님) |
| 투사체 이동·소멸 | 벽(플랫폼) 또는 플레이어와 **충돌하기 전까지 직선으로 계속 이동**; 충돌 시 소멸, 화면 밖 이탈 시에도 소멸 |
| 공격 색상 | **노란색**(패링·반사 가능) 또는 **빨간색**(패링 불가)으로 구분 |

#### 1.3.3 예외 처리

> - 감지 범위: 플레이어 크기의 **5배(= 플레이어 중심 반지름 5 world unit 원형)** / 투사체 히트박스 크기: 플레이어 크기의 **1/5배(= 반지름 0.2 world unit)** / 발사·공격 방향: 발사 시점의 플레이어 위치를 향함
> - 발사는 **별도의 근접 발사 거리 조건 없이** 플레이어가 감지 범위(5배) 내에 있는 동안 조준(1초) → 발사(1개) → 쿨타임(1초) 사이클로 반복; 투사체 자체는 벽/플랫폼/플레이어 충돌 전까지 직선 이동
> - 조준 도중(1초 경과 전) 플레이어가 감지 범위를 벗어나면 발사를 취소하고 대기 상태로 복귀 (구현 시 선택 적용; 기본은 취소 권장)
> - 투사체가 발사된 후 플레이어가 투사체 경로에서 벗어나도 투사체는 직선 유지 (발사 시점 방향 고정)
> - 발사 직선 경로 상에 벽(플랫폼)이 있으면 투사체는 벽에 충돌하여 소멸 (플레이어에게 도달하지 못함)
> - 플레이어가 패링으로 투사체를 반사한 경우, 반사된 투사체는 적에게 데미지를 입힘 (`player_design_doc.md` §5.4.2 참조)
> - 반사 투사체의 적 명중 데미지는 **명중 대상 무관 `플레이어 공격력 × 200%`로 통일** (원래 발사한 적/다른 적 모두 동일)
> - 여러 투사체가 동시에 플레이어에게 명중해도 피격 판정은 1회만 적용 (1피격 즉사 원칙)
> - 발사 타이밍 확정: **조준 1초 + 발사 후 쿨타임 1초** (사이클당 약 2초)
> - 원거리 적이 플랫폼 끝에서 추격하더라도 낭떠러지로 떨어지지 않음 (땅 끝에서 정지). §1.2.3 근거리 적과 동일하게 **`Raycast`/`Foothold.EndPoint` 기반 수동 발판 끝 감지로 구현**(MSW 기본 동작 아님, A-4)

### 1.4 보스

#### 1.4.1 기본 특성

- 보스 스테이지에만 등장하는 적으로, 일반 적보다 훨씬 높은 HP와 다양한 공격 패턴을 가짐
- 미리 정의된 **공격 패턴 풀(Pool)** 에서 패턴을 랜덤으로 선택하여 사용
- 근거리, 원거리 공격을 모두 사용할 수 있음

#### 1.4.2 그로기(기절) 시스템

체력 손실이 아닌 **그로기 게이지** 충전 방식으로 발동된다. 세키로의 "체간 시스템"과 유사한 구조를 참고하였다.

**그로기 게이지 기본 사양**

| 항목 | 내용 |
|---|---|
| 게이지 최대치 | 보스 **전체(최대) 체력의 1/5** 값 (현재 체력 기준 아님) |
| 게이지 충전 — 일반 공격 | 플레이어의 공격이 보스에게 명중한 데미지량만큼 충전 |
| 게이지 충전 — 근거리 패링 | 패링 발동 즉시 **일반 공격 명중 데미지(= `공격력 × 100%`)의 2배** 충전 (기본값 기준 **20**) |
| 게이지 충전 — 원거리 패링(투사체 반사) | 반사된 투사체가 보스에게 **실제로 명중**했을 때 **일반 공격 명중 데미지(= `공격력 × 100%`)의 2배** 충전 (기본값 기준 **20**, 반사 직후가 아닌 명중 시점 기준) |
| 게이지 자동 감소 | 전투 중 **초당 2%** 씩 감소 (보스 HP 수준과 무관하게 일정); 단 섬광탄 사용 시 5초간 자동 감소 정지(충전은 가능) — §3.2.2 참조 |
| 발동 조건 | 그로기 게이지 **100% 충전** 시 그로기 상태 진입 |
| 그로기 지속 시간 | **3초** |
| 그로기 효과 | 보스 이동 및 공격 완전 정지; 플레이어가 자유롭게 공격 가능 |
| 그로기 해제 | 지속 시간 종료 후 그로기 상태 해제, 패턴 재개 |
| 그로기 게이지 초기화 | 그로기 상태 진입 시 게이지 0으로 초기화; 이후 다시 충전하면 재발동 가능 |

> 참고 레퍼런스: [이나리 게임 데모 보스 (3:37~)](https://youtu.be/WjLkKjQilv0?t=217)

#### 1.4.3 공격 패턴 구조

| 항목 | 내용 |
|---|---|
| 패턴 선택 방식 | 정의된 패턴 풀에서 랜덤 선택 |
| 패턴 간 딜레이 | 각 패턴 종료 후 일정 대기 시간 후 다음 패턴 선택 |
| 패턴 색상 구분 | 각 패턴별로 공격 색상(노란색/빨간색) 설정 가능 |
| 패턴 개수 | 보스당 **3~5개**(검토 중); 시스템은 가변 개수의 패턴 풀을 지원하도록 구현 |
| 구체적 패턴 목록 | **미정** (추후 보스별로 별도 기획 문서 작성 필요; 구현 시 더미 패턴 2~3개로 플레이스홀더 권장) |

#### 1.4.4 예외 처리

> - 그로기 게이지는 **보스 전용**; 일반 적(근거리·원거리)에는 적용하지 않음
> - 그로기 상태 중에는 게이지 충전 및 자동 감소 모두 정지; 그로기 해제 후 게이지 0에서 다시 충전 시작
> - 그로기 상태 중 추가 공격이 들어와도 그로기 중첩 발동 없음
> - 그로기 게이지 자동 감소(초당 2%)는 보스 HP 수준과 무관하게 전투 중 항상 일정하게 적용 (그로기 상태 중 정지, 섬광탄 사용 시 5초간 정지)
> - 원거리 패링 시 그로기 게이지는 반사 투사체가 보스에게 실제로 명중한 시점에 충전; 반사 직후 또는 명중 실패 시(화면 밖 이탈 등)에는 충전되지 않음
> - 보스 사망(HP 0 도달) 시 그로기 상태이더라도 즉시 사망 처리
> - 보스 스테이지 재시작 시 그로기 게이지 0으로 초기화
> - 패턴 전환 도중 플레이어에 의해 보스가 사망하는 경우, 진행 중인 패턴 이펙트·투사체는 즉시 소멸 처리 (잔류 투사체로 인한 사후 피격 방지)
> - 보스 분노(Enrage) 페이즈는 **미도입**
> - 그로기 발동 시 결정타 연출은 **미도입**
> - 게이지 충전의 기준이 되는 "일반 공격 명중 데미지"는 `공격력 × 100%`이며, 공격력이 강화되면 패링 충전량(그 2배)도 비례하여 증가함 (의도된 설계)
> - **[MSW 구현 / 서버 권위(C-5)]** 그로기 게이지 수치·100% 발동·3초 그로기 지속·초당 2% 자동 감소·섬광탄 5초 감소 정지는 모두 **서버에서 권위적으로 처리**하고 클라이언트에는 `@Sync`로 동기화만 한다. 시간 측정은 **`ServerElapsedSeconds` 기준**이므로 클라이언트 슬로우 모션(`SetClientTimeScale`)의 영향을 받지 않는다 — 슬로우 중에도 게이지 자동 감소·그로기 지속 타이머는 정상(실제) 시간으로 흐른다.

## 2. 스테이지 구조

### 2.1 스테이지 종류 개요

| 종류 | 역할 | 클리어 조건 |
|---|---|---|
| 일반 스테이지 | 전투 스테이지 | 스폰된 모든 적 처치 |
| 상점 스테이지 | 메소 소모로 강화 구입 | 구매 완료 후 다음 스테이지 진행 |
| 보스 스테이지 | 보스 처치 후 다음 층 진행 | 보스 처치 |

---

### 2.2 층(Floor) 진행 방식

#### 2.2.1 전체 진행 구조 개요

**채택 방식**: 림컴(Limbus Company)의 거울 던전식 **테마팩 + 노드 기반 맵** 진행.

하나의 층(Floor)은 다음 흐름으로 진행되며, 보스 처치 후 다음 층에서 동일 구조가 반복된다.

```
[테마팩 선택 화면] → [노드 맵 진행: 일반 스테이지 N회 → 상점 → 보스] → [보스 클리어] → [다음 층 테마팩 선택 화면] → 반복
```

1. **테마팩 선택**: 각 층 시작 시, 진행할 **테마팩(Theme Pack)** 선택 화면이 표시됨 → §2.2.2
2. **노드 맵 진행**: 선택한 테마팩의 스테이지들이 노드 맵 형태로 펼쳐짐. 일반 스테이지 구간은 **분기 중 1개를 선택**하며 진행 → §2.2.3
3. **고정 구간**: 해당 층의 일반 스테이지를 규정 횟수만큼 완료하면 **상점 스테이지(고정)** → **보스 스테이지(고정)** 순으로 진행
4. **다음 층 전환**: 보스 처치 시 다음 층의 테마팩 선택 화면으로 전환

#### 2.2.2 테마팩 선택

| 항목 | 내용 |
|---|---|
| 표시 시점 | **각 층 시작 직전** (1층 시작 시, 그리고 매 보스 처치 후 다음 층 진입 시) |
| 선택 방식 | 표시된 테마팩 중 하나를 선택 (마우스 좌클릭) |
| 미플레이 테마만 표시 | 현재 게임 판에서 **이미 진행한 테마팩은 다시 표시되지 않음** |
| 본 개발 구현 범위 | 테마팩 **2종 구현** → 1층에서 2종 중 1택, 2층에서는 남은 1종만 표시(자동 1택 형태가 정상) |

> 테마팩 구성 (배경/지역 순서):
> - **테마 1 (확정)**: 오르비스 → 엘나스 → 폐광
> - **테마 2 (미정)**: 배경/지역 추후 확정 — 구현 시 자리표시자로 처리하고 시스템은 N종 테마를 지원하도록 작성

#### 2.2.3 노드 맵 및 스테이지 흐름

| 층 | 일반 스테이지 진행 횟수 | 상점 스테이지 | 보스 스테이지 |
|---|---|---|---|
| 1층 | 4회 | 1개 (고정) | 1개 (고정) |
| 2층 | 5회 | 1개 (고정) | 1개 (고정) |

- **스테이지 타입 순서는 고정**: `일반 스테이지 N회` → `상점 1개` → `보스 1개`
- **일반 스테이지 구간의 분기 선택**: 각 일반 스테이지 단계마다 **3개의 분기 노드 중 1개를 선택**하여 진행
  - 현재는 일반 스테이지 외 다른 노드 타입이 없으므로, 3개 분기 모두 일반 스테이지로 **임시 구현** (추후 이벤트·보물 등 노드 타입 추가 가능 구조로 작성)
- 일반 스테이지를 해당 층 규정 횟수(1층 4회·2층 5회)만큼 완료하면 **상점 노드(고정 선택)** → **보스 노드(고정 선택)** 순으로 자동 연결
- 층이 하나 올라갈 때마다 일반 스테이지 진행 횟수가 1회씩 증가
- **최대 구현 목표**: 2층 (추후 여유 시 5층까지 확장 예정)
- 5층 기준 예상 일반 스테이지 진행 횟수: 8회 (`4 + 1·(층수-1)` 공식 적용)

#### 2.2.4 예외 처리

> - 게임 시작 시(1층 테마팩 선택 표시 전) 이전 플레이 데이터가 잔존하지 않도록 플레이어 상태를 초기화 (단, 영구 강화 값은 유지 → §4.5 참조)
> - 이미 진행한 테마팩은 같은 게임 판 내에서 재등장하지 않도록 "사용된 테마" 목록을 게임 판 단위로 관리
> - 테마팩 2종만 구현된 현재, 2층 테마팩 선택 화면은 남은 1종만 표시됨 (선택지가 1개여도 선택 화면은 정상적으로 표시; 임시로 자동 진행 처리도 가능)
> - 일반 스테이지 분기 노드는 3개를 기본으로 하되, 맵 폭/구성에 따라 분기 수가 달라질 수 있음 (구현 시 데이터로 정의)
> - 분기 노드 선택 후 되돌아가 다른 분기를 다시 고를 수 없음 (전진 진행만 가능)
> - 상점·보스 노드는 분기 없이 고정 연결되므로 선택 UI는 "진행" 확인 형태로 단순화 가능
> - 노드 맵에서 어떤 스테이지로 진입하든, 진입 시 플레이어 목숨은 최대치로 갱신 (`player_design_doc.md` §1.4 참조)
> - 각 테마팩별 배경·적 구성 등 세부 특성은 추후 별도 기획 필요

### 2.3 일반 스테이지

#### 2.3.1 클리어 조건

- 스폰된 **모든 적을 처치**하면 스테이지 클리어
- 처치되지 않은 적이 1마리라도 남아 있으면 클리어 불가

#### 2.3.2 적 스폰 규칙

| 항목 | 내용 |
|---|---|
| 적 종류 | 근거리 적, 원거리 적 랜덤 조합 |
| 스폰 수량 | 배치된 스테이지에 따라 변동 (스테이지별 고정 구성) |
| 스폰 위치 | 스테이지 맵에 미리 지정된 스폰 포인트에서 발생 |
| 스폰 타이밍 | **일괄 스폰** (스테이지 진입 시 모든 적 동시 스폰); 추후 웨이브 방식 추가 가능성 있음 |

#### 2.3.3 사망 처리

- 플레이어 사망 시 **목숨 1 감소** 후 **현재 스테이지 재시작** (`player_design_doc.md` §1.4 목숨 갱신 규칙 참조)
- 목숨이 0인 상태에서 사망 시 **게임 오버** 처리 (타이틀 화면 또는 결과 화면으로 이동)

#### 2.3.4 레벨 디자인 방식

- 미리 제작된 **최소 20개 이상**의 일반 스테이지 맵 중 하나가 **랜덤 선택**되어 발생
- **[대안]** 미리 제작된 스테이지 구성 조각(Chunk)들이 무작위로 조립되는 하이브리드 방식도 검토 가능
  - 참고: [Dead Cells 레벨 디자인 방식](https://deepnight.net/tutorial/the-level-design-of-dead-cells-a-hybrid-approach/)

> **[맵 직접 제작 시 정의해야 할 항목 — MSW 구현 기준(C-4)]**
> 일반 스테이지 맵을 직접 제작하므로, **맵마다** 다음을 확정해야 한다:
> - **`TileMapMode = 0`(MapleTile)** 로 생성 (Maker Hierarchy에서 확인/전환; 횡스크롤+중력+Foothold)
> - **맵 ID·네이밍 규칙**: 랜덤 풀 선택·직전 맵 중복 방지·테마팩 필터를 위한 일관된 식별자 체계 (예: `normal_{theme}_{nn}`) + 테마 태그(오르비스/엘나스/폐광 등)
> - **발판(Foothold) 배치** 및 **플레이어 시작 스폰 위치**
> - **적 스폰 포인트**: 각 지점의 좌표(world unit) + 배치할 적 종류(근거리/원거리) + 수량 — 스테이지별 고정 구성(§2.3.2)
> - **카메라/맵 범위**: PC 12.8×7.2 / 모바일 9.6×5.4 world unit 뷰포트 기준으로 시작 화면 구도가 적절한지 (1 unit = 100px)
> - **SectorConfig 등록**: 제작한 모든 맵을 `Global/SectorConfig.config`의 `entries`에 `map://{id}` 형태로 등록
> - **분기 노드 ↔ 맵 배정 데이터**: 일반 스테이지 3분기 각각에 풀에서 어떤 맵이 배정되는지 정의 (§2.2.3)
> - (참고) 적은 `RigidbodyComponent` 필요, 낭떠러지 정지는 수동 구현(§1.2.3) — 발판 끝 좌표를 AI가 참조할 수 있도록 배치 시 고려

#### 2.3.5 예외 처리

> - 랜덤 선택 시 직전에 플레이한 맵과 동일한 맵이 연속으로 선택되지 않도록 중복 방지 로직 적용 권장
> - 스테이지 재시작 시 맵 구조는 유지하되, 적의 위치 및 스폰은 초기 상태로 재설정
> - 스테이지 재시작 시 해당 스테이지 내에서 획득한 임시 보상(메소 등)은 초기화 (스테이지 클리어 후 지급 방식이므로 재시작 시 이월 없음)
> - 스테이지 진입 시 능력 사용 횟수는 일반 스테이지 기준(1회)으로 초기화 (`player_design_doc.md` §4.4 참조)
> - 스테이지 진입(및 재시작) 시 플레이어 목숨은 최대치로 갱신 (`player_design_doc.md` §1.4 참조)
> - 노드 맵의 일반 스테이지 분기(3개) 각각에는 §2.3.4의 맵 풀에서 선택된 맵이 배정됨; 분기 선택 시 해당 맵으로 진입 (§2.2.3 참조)
> - 스테이지 내 모든 적이 처치되었으나 클리어 판정이 발생하지 않는 버그를 방지하기 위해, 스폰된 적 수와 처치된 적 수를 카운터로 관리하고 두 값이 일치 시 클리어 처리
> - **[MSW 구현 — 단일 초기화 함수(A-5)]** 스테이지 재시작은 **하나의 초기화 함수**로 일괄 처리하여 부분 초기화로 인한 상태 잔류 버그를 방지한다. 함수가 수행할 항목: (1) 잔존 적 엔티티 전부 소멸, (2) 비행 중 투사체 전부 소멸, (3) **슬로우 모션 상태 강제 해제**(클라 `SetClientTimeScale(1.0)` + 적 속도 배율·`PlayRate` 원복 → `player_design_doc.md` §3.2.5 참조), (4) 스폰/처치 카운터 초기화, (5) 적 재스폰(초기 위치), (6) 플레이어 위치·목숨(최대치)·능력 사용 횟수·스테이지 내 임시 메소 초기화.

### 2.4 상점 스테이지

#### 2.4.1 구입 품목 목록 및 초기 가격

| # | 품목 종류 | 효과 | 초기 가격(메소) | 발생 수량 | 필수 발생 여부 |
|---|---|---|---|---|---|
| 1 | **최대 목숨 추가** | **최대 목숨 +1** (구입 즉시 현재 목숨도 +1; 목숨 회복이 아님) | **100** | 1개 | ✅ 필수 |
| 2 | 스탯 강화 | 선택된 스탯 1종 강화 | **50** | 1~2개 | ✅ 최소 1개 필수 |
| 3 | 능력 획득/강화 | 보유 능력 강화 또는 새 능력 획득 | **150** | 1개 | ✅ 필수 |
| 4 | 일회성 아이템 | 연막탄·섬광탄 등 | **40** | 1~2개 | ✅ 최소 1개 필수 |

- 한 번에 표시되는 **최대 구입 품목 수: 5개**
- 가격은 갱신 시 2배씩 상승 (§2.4.4 참조)

> ⚠️ **용어 정정**: 기존 "목숨 회복"은 폐기. 목숨은 스테이지 진입 시마다 최대치로 갱신되므로 회복 개념이 성립하지 않으며, 상점 품목은 **최대 목숨을 +1 늘리는** "최대 목숨 추가"로 정의한다 (`player_design_doc.md` §1.4 참조).

#### 2.4.2 구매 조작 및 UI 흐름

| 단계 | 내용 |
|---|---|
| 품목 선택 | 구입할 품목 아이콘을 **마우스 좌클릭** |
| 구매 확인 | 좌클릭 시 **"정말로 구입하겠습니까?"** 확인 팝업 표시 (확인/취소) |
| 구매 확정 | 확인 시 메소 차감 및 효과 적용; 동일 종류 품목이 가격 2배로 즉시 갱신 표시 (§2.4.4) |
| 상점 종료 | 상점 UI의 **나가기 버튼** 클릭 시 상점을 닫고 **다음 노드(보스 스테이지)로 진행** (`§2.2.3` 노드 흐름 참조) |

> 나가기 버튼은 구매를 한 번도 하지 않아도 누를 수 있음 (구매는 선택 사항). 다만 능력/아이템 등 필수 발생 품목은 "표시"가 필수이지 "구매"가 필수는 아님.

#### 2.4.3 스탯 강화 품목 상세

- 강화 가능한 스탯: `공격력`, `목숨`(최대 목숨), `최대 대시 횟수`, `대시 회복 시간`, `최대 아이템 보관 수`
  - (`공격 범위`는 스탯 미도입 확정으로 강화 대상에서 제외 — `player_design_doc.md` §1.1 참조)
- 표시 방식: **랜덤으로 스탯 강화 품목이 상점에 발생**하는 방식

#### 2.4.4 능력 획득/강화 품목 상세

- 표시 내용: 현재 보유 능력을 강화하는 품목 **또는** 새로운 능력 품목 중 하나가 발생
- 능력 교체 선택 시 확인 UI 표시 필요 (`player_design_doc.md` §4.7 참조)

#### 2.4.5 품목 갱신 규칙

- 구입한 품목과 **동일한 종류**의 품목이 **즉시 갱신**되어 다시 표시됨
- 갱신된 품목의 **가격은 직전 가격의 2배**로 상승 (갱신할 때마다 매번 2배; 예: 스탯 강화 50 → 100 → 200 → …)
- **가격 상한선 없음** (무한 갱신 가능); 단, 메소 캡(4,294,967,295)을 넘는 가격 표기 시 정수 오버플로우가 발생하지 않도록 처리

#### 2.4.6 예외 처리

> - 메소 잔액이 부족한 경우 구입 불가 처리; 확인 팝업 단계 또는 클릭 시점에 UI 피드백 표시 (`player_design_doc.md` §6.5 참조)
> - **최대 목숨 추가** 품목은 회복이 아니라 최대치 증가이므로, 목숨이 가득 찬 상태에서도 항상 구입 가능 (구입 불가/비활성화 조건 없음)
> - 최대 아이템 보관 수(3칸)가 가득 찬 상태에서 일회성 아이템 구입 시 버리기/교체 UI 표시 (`player_design_doc.md` §1.3 참조)
> - 능력 없는 상태에서 능력 강화 품목만 발생하는 경우를 방지하기 위해, 능력 미보유 시 능력 품목은 반드시 **새 능력 획득** 품목으로 발생
> - 상점 스테이지 진입 시 플레이어 사망 처리는 없음 (전투 없는 스테이지); 진입 직후 적 등장 로직이 실행되지 않도록 분기 처리 필요
> - 갱신 가격 상한선은 **미설정** (무한 갱신 가능); 갱신 횟수에 따라 가격이 계속 2배씩 상승하되 메소 캡 한도 내에서 오버플로우 방지
> - 나가기 버튼을 누르기 전까지는 상점에 머물며 반복 구매 가능; 나가기 클릭으로만 다음 노드로 진행되도록 트리거 처리

---

### 2.5 보스 스테이지

#### 2.5.1 클리어 조건

- **보스를 처치**하면 스테이지 클리어
- 클리어 후 보스 클리어 보상 지급 → [3.3 보스 스테이지 보상](#33-보스-스테이지-보상) 참조

#### 2.5.2 사망 처리

- 일반 스테이지와 동일: 목숨 1 감소 후 보스 스테이지 재시작
- 목숨 0 상태에서 사망 시 게임 오버

#### 2.5.3 예외 처리

> - 보스 스테이지 재시작 시 보스 HP 및 그로기 게이지 전부 초기화 (§2.3.5 **단일 초기화 함수** 규칙을 보스 스테이지에도 동일 적용 — 잔류 투사체·이펙트 소멸 및 슬로우 모션 상태 해제 포함)
> - 보스 스테이지 진입 시 능력 사용 횟수는 보스 스테이지 기준(2회)으로 초기화 (`player_design_doc.md` §4.4 참조)
> - 보스 스테이지 진입(및 재시작) 시 플레이어 목숨은 최대치로 갱신 (`player_design_doc.md` §1.4 참조)
> - 보스 처치 직후 잔류하는 투사체 및 이펙트 즉시 소멸 처리 (§1.4.4 참조)
> - 보스 처치 및 보상 수령 완료 후 **다음 층 테마팩 선택 화면**으로 전환 (마지막 층이면 게임 클리어 화면) (§2.2.2 참조)
> - 보스 스테이지는 고정된 전용 맵 1개를 사용하는 것을 기본으로 함 (복수 맵 사용 여부 추후 결정)

## 3. 스테이지 보상

### 3.1 보상 수령 UI 규칙

- 보상 아이콘을 **마우스 좌클릭**하여 개별 수령
- 일부 보상은 받지 않고 **버리는 선택지**도 존재
- 능력 보상의 경우 교체 전 확인 UI 표시 (`player_design_doc.md` §4.7 참조)

---

### 3.2 일반 스테이지 보상

#### 3.2.1 보상 목록

| 보상 종류 | 수령 방식 | 획득 확률 |
|---|---|---|
| 메소 (재화) | 스테이지 클리어 시 자동 확정 지급 (적 처치 즉시 드롭 아님) | **100%** (고정) |
| 스탯 강화 (해당 판 내 임시) | 선택 수령 | **50%** |
| 일회성 아이템 (연막탄·섬광탄 등) | 선택 수령 | **70%** |
| 능력 | 선택 수령 | **15%** |

> 각 보상의 등장 확률은 보상별로 독립 판정한다 (한 스테이지 클리어 시 여러 보상이 동시에 제시될 수 있음). 메소는 항상 확정 지급되며 나머지는 위 확률로 등장 여부를 결정한다.

#### 3.2.2 일회성 아이템 종류

| 아이템 이름 | 효과 | 비고 |
|---|---|---|
| 짱돌 | 사용 시 투사체 공격 발사; 명중 시 **플레이어 공격력 × 300%** 데미지 | 투사체형 공격 아이템 |
| 연막탄 | 사용 시 **플레이어 위치 중심으로 플레이어 크기 3배 범위(= 반지름 3 world unit 원형)**에 은신 영역을 **5초간** 활성화 | 일반 적만 감지/조준 해제; 보스는 은신 무시 |
| 섬광탄 | 사용 시 **화면에 보이는 모든 일반 적**에게 **3초간 기절**(공격·이동 모두 정지) 적용 | 보스는 기절 면역; 대신 그로기 게이지 감소만 5초 정지 (충전은 가능) |
| (추가 예정) | - | 추후 기획 확장 시 추가 |

> 🔧 위 3종은 효과 확정. 그 외 아이템은 추후 기획 확장 시 본 표에 항목 추가.

> 최대 보관 수(3칸)는 `player_design_doc.md` §1.1을 따른다. **일회성 아이템 사용 조작**: 아이템은 보관 슬롯 **1~3번 숫자키**에 대응하며(슬롯 자리 변경은 미구현), 해당 숫자키를 눌러 **활성화 상태**로 만든 뒤 **좌클릭** 시 커서 방향으로 사용/즉시 적용된다. 숫자키를 다시 누르면 비활성화된다. (상세 규칙은 `player_design_doc.md` §7 참조)

**일회성 아이템 예외 처리**

> - **공통**: 아이템 사용 효과는 플레이어가 발생시키는 것이므로 적의 패링 대상이 아니다 (적은 패링하지 않음).
> - **공통(일회성)**: 모든 일회성 아이템은 사용 시 1회 소모되어 사라진다. 재사용·중첩·지속 갱신 개념 없음.
> - **공통(시간 기준)**: 아이템 효과 지속 시간(연막탄 5초, 섬광탄 일반 적 기절 3초, 섬광탄 보스 게이지 감소 정지 5초)은 **실제 시간(Wall-clock) 기준**으로 측정하며 슬로우 모션의 영향을 받지 않는다. 짱돌 투사체도 플레이어 생성물이므로 슬로우 영향 없이 정상 속도로 이동한다 (`player_design_doc.md` §7.4 참조).
> - **공통(사용 제한)**: 전투 스테이지(일반·보스)에서만 사용 가능하며, 상점·보상 UI 중에는 사용 불가 (`player_design_doc.md` §7.4 참조).
> - **짱돌**: 투사체 데미지는 `공격력 × 300%`(기본값 기준 30). **발사 방향 = 커서 방향, 투사체 속도·히트박스 = 적 투사체 사양 재사용, 충돌/소멸 규칙도 적 투사체와 동일**(벽/플랫폼/적 충돌 시 소멸, 적 명중 시에만 데미지). (`§1.3.2` 적 투사체 사양 참조)
> - **짱돌(보스)**: 보스 명중 시 그로기 게이지는 "명중 데미지량만큼" 충전(일반 공격과 동일 규칙, §1.4.2). 별도 배수 없음.
> - **연막탄**: 활성화 **시점의 플레이어 위치에 고정**된, 플레이어 크기 3배 범위의 은신 영역을 5초간 생성(플레이어를 따라다니지 않음). 은신 지속 동안 일반 적(근거리·원거리)은 플레이어를 감지/조준할 수 없으며, **이미 추격·조준 중이던 적도 즉시 해제**되어 대기 상태로 복귀. 일회성이므로 재사용으로 인한 갱신·중첩 처리 불필요. **보스는 예외적으로 은신을 무시**하고 정상적으로 패턴을 진행한다.
> - **섬광탄(일반 적)**: 카메라 화면(뷰포트)에 보이는 **모든 일반 적**에게 3초 기절(이동·공격 완전 정지) 적용. 기절 중인 적은 공격 판정도 발생하지 않음.
> - **섬광탄(보스 예외)**: **보스에게는 3초 기절이 적용되지 않는다**(보스는 정상적으로 이동·공격·패턴 진행). 대신 **보스의 그로기 게이지가 5초간 자동 감소(초당 2%)만 정지**되며, 그 5초 동안에도 **플레이어 공격에 의한 게이지 충전은 정상적으로 이루어진다**(감소 없이 충전만 누적 가능). 5초 경과 후 자동 감소 재개.

#### 3.2.3 예외 처리

> - 메소는 적 처치 즉시 드롭되지 않으며, 스테이지 클리어 시 한꺼번에 자동 지급됨; 별도 수령 UI 없음
> - 메소 지급 공식 (임시): **일반 스테이지** — `일반 스테이지 클리어 수 × 10` / **보스 스테이지** — `보스 스테이지 클리어 수 × 150` (`player_design_doc.md` §6.3 참조)
> - 스탯 강화·아이템·능력 보상은 클리어 후 보상 UI 화면에서 선택적으로 수령
> - 능력 보상 제시 시 "획득하지 않음(버림)" 선택지도 함께 표시 (`player_design_doc.md` §4.7 참조)
> - 최대 아이템 보관 수 초과 시 아이템 획득 선택 불가; 버리기/교체 UI 표시 (`player_design_doc.md` §1.3 참조)
> - 보상 UI가 열려 있는 동안 플레이어 이동 및 전투 입력 비활성화 처리 필요
> - 보상 UI 없이 다음 스테이지로 강제 진행하는 버그를 방지하기 위해, 모든 보상 처리 완료 후 다음 스테이지 진행 트리거 활성화

---

### 3.3 보스 스테이지 보상

#### 3.3.1 보상 목록

- 보상 종류는 일반 스테이지 보상 목록과 동일
- 단, 보스 스테이지 클리어 시 **보상 목록 전체를 획득 가능한 상태로 제시**
  - 일반 스테이지는 확률에 따라 일부 보상만 제시되지만, 보스 스테이지는 전체 보상 목록이 항상 제시됨
- 수령 방식은 동일: 마우스 좌클릭으로 개별 수령, 버리기 선택지 존재

#### 3.3.2 예외 처리

> - 보스 스테이지 보상 UI 처리 규칙은 일반 스테이지 보상 예외 처리와 동일하게 적용
> - 보스 스테이지 보상 수령 완료 후 다음 층(Floor) 진입 또는 게임 클리어 화면으로 전환

## 4. 게임 클리어 보상 및 메타 진행

### 4.1 정의

- 모든 층을 클리어하면 **영구 강화 점수**를 획득
- 획득한 점수는 게임 판 외부의 **메타 진행 트리**에서 소모하여 **영구 스탯 증가** 또는 **능력 사전 획득/강화** 가능
- 영구 강화 내용은 이후 모든 게임 플레이에 기본 적용됨

---

### 4.2 영구 스탯 강화 목록

| 강화 대상 스탯 | 비고 |
|---|---|
| 공격력 | - |
| 목숨 | - |
| 최대 대시 횟수 | - |
| 대시 회복 시간 | 단축 방향으로 강화 |
| 최대 아이템 보관 수 | - |

> `공격 범위`는 스탯 **미도입 확정**(`player_design_doc.md` §1.1)이므로 영구 강화 대상에서 제외한다.

---

### 4.3 능력 사전 획득/강화 목록

- 기획된 능력 목록 전체가 강화 대상으로 존재
- 각 능력에 연계된 **단계별 강화 항목**이 존재 (`player_design_doc.md` §4.5 강화 방향 참조)

---

### 4.4 점수 획득 기준

- **확정**: `일반 스테이지 클리어 수 + (보스 스테이지 클리어 수 × 3)`

---

### 4.5 예외 처리

> - 게임 오버 시 영구 강화 점수 미지급 (클리어 시에만 지급)
> - 영구 강화는 게임 판과 독립적으로 저장됨; 게임 오버 또는 클리어 후에도 유지
> - 메타 진행 트리 UI 및 점수 소모 방식은 별도 기획 문서 작성 필요
> - 영구 강화 점수 누적 상한선: **65,535** (unsigned short 크기 기준)
> - 영구 강화로 인해 기본값이 변경된 스탯은 게임 시작 시 변경된 값으로 초기화되어야 함 (예: 영구 강화로 목숨 기본값이 5 → 6이 된 경우, 매 게임 시작 시 목숨 6으로 시작)
